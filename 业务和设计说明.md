# 盛京医疗联盟儿童心理健康服务平台演示程序业务说明,仅作为参考
## 一、业务概述
本演示程序是盛京医疗联盟旗下**0-18岁儿童青少年心理健康专属数字化服务平台**的功能原型，核心目标是完整演示"**科普教育前置→心理测评识别→绿色预约诊疗→评价反馈沉淀**"的全流程业务闭环，重点验证C端用户操作与后台管理数据的**毫秒级实时联动**特性。

平台以家长为核心服务对象，整合联盟内儿童精神科专家资源，解决儿童心理问题识别难、优质号源预约难、服务流程不透明三大痛点。演示程序采用前后端一体架构，内置模拟数据，无需对接真实系统即可完整展示所有核心业务逻辑与交互效果。

## 二、业务详细说明
### （一）整体演示入口设计
1. **统一演示首页**：作为程序启动后的唯一入口，页面简洁展示平台名称与两个核心入口按钮：
   - 「C端家长服务端」：点击进入H5响应式移动端演示界面
   - 「后台管理端」：点击进入PC端管理系统演示界面
   - 页面底部添加一行小字说明："本程序为功能演示原型，所有数据均为模拟数据"
2. **入口跳转逻辑**：点击对应按钮直接跳转至目标系统首页，无需登录验证（模拟主应用已完成身份认证）

### （二）C端家长服务端业务说明（H5移动端）
#### 1. 首页（核心导航页）
- **业务目标**：聚合所有核心服务入口，快速引导用户完成目标操作
- **核心功能**：
  - 顶部展示平台名称与常驻紧急求助按钮（红色醒目，仅做展示，无需实现拨号功能）
  - 中部2×2布局展示四大核心功能卡片：心理科普、门诊预约、我的测评、我的预约
  - 下方展示推荐内容区：2条热门科普视频+3篇精选科普文章+4位推荐儿童心理专家
  - 点击任意内容或功能卡片直接跳转至对应页面

#### 2. 儿童心理精神科普知识模块
- **业务目标**：为家长提供专业、易懂的儿童心理健康知识，建立初步信任
- **核心功能**：
  - 内容列表页：支持按内容类型（全部/图文/视频）切换，按分类筛选（情绪行为、学习困难、亲子关系、发育问题、睡眠问题）
  - 内容卡片：展示封面、标题、作者、发布时间、阅读/播放量、收藏按钮
  - 内容详情页：展示完整图文或视频内容，底部显示"有相关困扰？立即预约对应专家"引导栏，自动推荐3位擅长该领域的医生
  - 收藏功能：点击收藏按钮可实时收藏/取消收藏，收藏状态在所有页面同步更新

#### 3. 儿童心理测评模块
- **业务目标**：帮助家长初步识别儿童心理问题，为后续诊疗提供参考
- **核心功能**：
  - 量表列表页：支持双维度筛选（年龄段：0-3岁/3-6岁/6-12岁/12-18岁；问题类型：注意力/情绪/行为/社交/学习）
  - 测评流程：选择量表→选择对应儿童就诊人→逐题答题（每题一页，显示进度条）→提交生成报告
  - 测评报告页：展示综合得分、风险等级（低/中/高）、简单解读与建议，高风险结果自动弹出红色预警弹窗
  - 诊疗衔接：报告底部提供"一键预约推荐专家"按钮，直接跳转至对应医生详情页
  - 样例要求：内置3个简单量表，每个量表5-10道单选题即可，重点演示流程而非专业准确性

#### 4. 门诊预约绿色通道模块
- **业务目标**：实现家长代儿童快速预约专家号，解决号源不透明、预约流程繁琐问题
- **核心功能**：
  1. **就诊人管理**：
     - 支持新增、编辑、删除儿童就诊人，一个账号最多添加5个
     - 就诊人必填字段：姓名、身份证号、出生日期、性别、就读年级、与本人关系、联系电话、紧急联系人、紧急联系电话
     - 系统自动根据出生日期计算儿童年龄，显示"儿童"标识
  2. **三步预约流程**：
     - 第一步：选择医生与时段。医生列表默认展示擅长儿童心理的医生，点击医生进入详情页，查看未来7天出诊时间表，实时显示剩余号源，点击可预约时段进入下一步
     - 第二步：选择就诊人并填写预问诊。选择已添加的儿童就诊人，填写儿童专属预问诊问卷（6道题：①本次就诊主要问题 ②症状持续时间 ③是否曾在其他医院就诊 ④主要行为表现 ⑤生长发育情况 ⑥近半年家庭环境是否有重大变化）
     - 第三步：确认预约。展示预约信息汇总与就诊须知，点击提交完成预约
  3. **我的预约**：
     - 按状态展示订单列表（待就诊/已完成/已取消）
     - 订单详情页展示完整预约信息、就诊人信息、预问诊结果
     - 支持取消预约（仅就诊前24小时可取消，需二次确认）

#### 5. 个人中心模块
- **业务目标**：聚合用户个人相关功能与数据
- **核心功能**：
  - 顶部展示用户头像与姓名（模拟主应用同步数据）
  - 功能入口：我的就诊人、我的收藏、我的测评、我的预约、我的评价
  - 点击任意入口跳转至对应页面

#### 6. 我的评价模块
- **业务目标**：收集用户就诊反馈，优化服务质量
- **核心功能**：
  - 待评价列表：展示已完成但未评价的订单
  - 评价功能：对医生进行星级评分（1-5星），填写文字评价
  - 意见留言：提交平台使用意见与建议
  - 已评价列表：展示历史评价记录

### （三）后台管理端业务说明（PC端）
#### 1. 实时运营仪表盘
- **业务目标**：让运营人员实时掌握平台整体运营状态
- **核心功能**：
  - 核心数据卡片：展示今日/本周/本月访问量、科普阅读量、测评完成数、预约订单数
  - 趋势图：展示近7天预约量与测评量变化趋势
  - 实时动态列表：滚动展示最近20条C端操作记录（预约成功、取消预约、完成测评、收藏内容）
  - 待办事项：展示待审核内容、待处理预约、即将到期随访
  - 所有数据实时更新，无需手动刷新

#### 2. 科普知识内容管理模块
- **业务目标**：实现科普内容的全生命周期管理
- **核心功能**：
  - 内容列表：展示所有图文/视频内容，支持按状态（草稿/待审核/已发布/已下架）筛选
  - 内容操作：新增、编辑、删除、发布、下架、预览
  - 分类管理：维护科普内容5大一级分类，支持新增二级分类
  - 审核流程：运营创建内容→提交审核→点击发布→C端立即可见

#### 3. 医生/医院管理模块
- **业务目标**：维护联盟内医疗资源信息
- **核心功能**：
  - 医生管理：新增、编辑、删除医生信息，配置医生姓名、职称、所属医院、擅长领域、头像等
  - 医院管理：新增、编辑、删除联盟医院信息，配置医院名称、地址、联系电话等

#### 4. 实时号源管理模块
- **业务目标**：实现医生排班与号源的精细化管理，确保号源数据实时准确
- **核心功能**：
  - 排班配置：支持两种视图切换（周视图/月视图），点击日期单元格新增排班，设置时段与号源数量
  - 批量排班：支持设置每周重复排班（如每周一、三、五上午出诊）
  - 号源实时监控：展示所有医生各时段总号源、剩余号源、已预约数，剩余号源≤3时标红预警
  - 号源调整：手动增加/减少号源数量，调整后实时同步至C端
  - 停诊管理：设置医生停诊时间，自动标记对应时段不可预约

#### 5. 预约订单管理模块
- **业务目标**：实现预约订单的全流程管理
- **核心功能**：
  - 订单列表：实时展示所有预约订单，C端提交后立即显示
  - 订单筛选：按订单状态、预约时间、医生、医院、儿童年龄段筛选
  - 订单详情：展示完整订单信息、就诊人信息、预问诊结果
  - 订单操作：确认预约、取消预约、改期预约、发送提醒
  - 导出功能：支持将订单数据导出为Excel文件

#### 6. 心理测评管理/量表分析模块
- **业务目标**：管理心理测评量表，统计分析测评结果
- **核心功能**：
  - 量表管理：查看已上线的心理量表信息
  - 测评结果统计：按量表、年龄段、风险等级统计测评完成情况
  - 趋势分析：展示近30天各量表测评量变化趋势
  - 高危结果列表：展示所有高风险测评结果，标注儿童姓名、测评时间、风险等级

### （四）核心业务规则与实时联动机制（必须严格实现）
1. **预约-号源-订单实时联动**：
   - C端提交预约→后端原子性操作：对应时段号源-1、生成新订单→通过WebSocket推送更新至C端医生详情页、后台号源监控页、后台订单列表页
   - C端取消预约→后端原子性操作：对应时段号源+1、订单状态更新为已取消→实时同步至所有客户端
2. **内容发布实时联动**：
   - 后台点击发布内容→后端更新内容状态→推送更新至C端→C端刷新科普列表立即显示新内容
3. **收藏操作实时联动**：
   - C端收藏/取消收藏→后端更新收藏记录→实时同步至我的收藏页与后台内容列表收藏数
4. **预约限制规则**：
   - 同一儿童同一医生7天内最多预约2次
   - 就诊前24小时可免费取消预约，24小时内不可取消
   - 号源每日0点自动释放次日号源
5. **儿童专属规则**：
   - 所有内容、量表、预问诊均针对0-18岁儿童设计
   - 医生列表默认优先展示擅长儿童心理的专家
   - 儿童就诊人必须填写紧急联系人信息

### （五）演示程序特殊要求
1. **用户模拟**：无需实现独立注册登录系统，默认模拟一个已登录的家长用户
2. **模拟数据**：内置完整模拟数据，包括10篇科普内容（6篇图文+4个视频）、10位儿童心理医生、3个心理量表、未来14天的排班与号源
3. **拟真页面**：所有页面采用拟真设计，无需添加业务介绍、说明文字或注释，仅保留功能操作元素
4. **启动要求**：提供完整的package.json依赖配置与start.bat启动脚本，执行npm install后运行npm start即可启动程序，自动打开浏览器访问演示首页
5. **错误处理**：添加基础错误处理机制，确保演示过程中不会出现崩溃或白屏
